# Handoff — مراجعة المشروع الشاملة (جلسة متعددة)

هذا الملف **مؤقت** — احذفه عند انتهاء العمل ودمج كل شيء. الغرض منه نقل
السياق لجلسة Claude جديدة إن انتهت هذه الجلسة قبل إكمال المهمة.

## السياق العام
مراجعة شاملة ومتكررة لتطبيق "مدار" طلبها المالك (عدة جولات: مشكلة الكيبورد
في المذكرات، الخلفية، تجربة التحرير، الأداء، ثم مراجعة أعمق). كل جولة سابقة
اكتملت ودُمجت في `main` بنجاح (آخر دمج: commit `907b1e2`).

الجولة الحالية: المالك طلب تنفيذ بقية الاقتراحات **بدون** إضافة ميزة
المزاج للنموذج — بل **حذف المزاج نهائياً من كل التطبيق**.

## ✅ منجز في هذه الجلسة (غير مدفوع بعد)

**حذف المزاج بالكامل** (تم واختُبر بالبناء `npm run build` ✓ نجح):
- `src/lib/types.ts`: حذف حقل `mood` من `JournalEntry` وحذف `MOOD_LABELS`.
- `src/lib/dayAggregator.ts`: حذف `mood` من `DaySummary`.
- `src/lib/insights.ts`: حذف توصية "الشراء العاطفي" (مزاج مقابل صرف).
- `src/app/stats/page.tsx`: حذف قسم "مزاجك في المذكرات" بالكامل (الرسم البياني).
- `src/components/dashboard/WeeklyWrap.tsx`: حذف إيموجي "المزاج السائد".
- `src/components/journal/JournalEntryCard.tsx`: حذف عرض أيقونة المزاج.
- `src/components/day/DayView.tsx`: حذف عرض المزاج.

**بونص أثناء نفس التعديل** (كان بند منفصل في المراجعة، دُمج لأنه بنفس الملفات):
- `src/lib/dayAggregator.ts` + `src/components/day/DayView.tsx`: `DayView` كان
  يعرض **أول مذكرة فقط** لليوم (`journalEntries.find`)؛ الآن يعرض **كل**
  مذكرات اليوم (`journalEntries.filter` → مصفوفة، مع عرضها جميعاً في القسم).

**تحقّق:** `npm run build` نجح بعد كل هذه التعديلات (لم يُشغَّل التطبيق فعلياً
بعد بـ Playwright — يُفضَّل التحقق قبل الدمج النهائي، خصوصاً DayView
بمذكرتين بنفس اليوم).

## ❌ متبقٍّ من قائمة المراجعة (لم يُبدأ بعد)

بالأولوية كما اتفقنا مع المالك:

1. **(الأهم) النسخة الاحتياطية بصور حقيقية:** حالياً `BackupCard.tsx` →
   `exportJson()` يستخدم `snapshot()` مباشرة، والذي يحمل **روابط** Firebase
   Storage للصور (بعد `feat(sync): نقل تخزين الوسائط...`) لا الصور نفسها.
   المطلوب: عند التصدير، نزّل كل صورة/صوت من رابطه وحوّله base64 قبل
   التضمين في الملف (مع مؤشر تقدّم بصري أثناء التنزيل — قد يكون كثيراً).
   ملفات ذات صلة: `src/components/settings/BackupCard.tsx`,
   `src/lib/sync.ts` (فيه `isStorageUrl`, `hydrateCloudPhotos` للمرجعية).

2. **المعاملات المتكررة تتولّد من الصفحة الرئيسية والمالية فقط:**
   `runRecurring()` يُستدعى في `useEffect` بـ `src/app/page.tsx` و
   `src/app/finance/page.tsx`. المطلوب نقله لمستوى التطبيق
   (`src/app/layout.tsx` أو مكوّن client يُركَّب هناك) ليعمل من أي صفحة.

3. **اختصارات PWA:** `public/manifest.webmanifest` بلا `shortcuts`.
   أضِف اختصارين: "مصروف سريع 💸" (رابط يفتح نموذج، يحتاج تنسيق مع
   `src/app/page.tsx` الذي فيه `quickExpense` state) و"مذكرة جديدة ✍️"
   (رابط لـ `/journal?new=1` مثلاً، مع قراءة query param في
   `src/app/journal/page.tsx` لفتح النموذج تلقائياً).

4. **مربع كتابة المساعد لا يتمدد:** `src/app/assistant/page.tsx` —
   `<textarea rows={1} ...>`. طبّق نفس تقنية auto-grow المستخدمة في
   `src/components/journal/JournalForm.tsx` (`useEffect` يضبط
   `style.height` حسب `scrollHeight`).

5. **مفاتيح الاحتفال تتراكم للأبد:** `src/app/page.tsx` —
   `localStorage.setItem(`madar-celebrated-${todayStr}`, "1")` بلا تنظيف.
   أضِف تنظيفاً للمفاتيح الأقدم من ~30 يوماً عند كل فتح (امسح كل مفتاح
   `madar-celebrated-*` غير آخر ٣٠ يوماً).

6. **وتيرة القراءة لأول كتاب فقط:** `src/components/reading/ReadingPace.tsx`
   يُستدعى من `src/app/reading/page.tsx` بـ
   `books.find((b) => b.status === "أقرأ")` — كتاب واحد. المطلوب: لكل
   الكتب الجارية (`filter` بدل `find`)، وعرض بطاقة وتيرة لكل واحد.

**لم يُطلب تنفيذها بعد (أولوية دنيا، اذكرها لو سُئلت):**
- محلّل CSV البنكي لا يدعم فواصل داخل علامات تنصيص (`bankParser.ts`).
- كل عملية غير معروفة تُصنَّف "أساسيات" تلقائياً بدل "غير مصنف" (قد يكون مقصوداً).

## حالة Git

- الفرع الحالي: `main` (التعديلات أعلاه **غير مضافة للـ commit بعد**، وغير
  مدفوعة).
- الفرع المخصّص لهذه المهمة حسب تعليمات الجلسة: `claude/project-review-bugs-busauw`
  (موجود محلياً وعلى origin، وكان آخر مرة مطابقاً لـ `main` عند نقطة `907b1e2`).
- ⚠️ **تنبيه مهم:** دليل المشروع (`CLAUDE.md`) يقول "اكتب مباشرة على main"
  كقاعدة عامة، لكن تعليمات هذه الجلسة تحدّد فرعاً مخصّصاً. المالك في هذه
  المحادثة كان يطلب صراحةً "ادمج" بعد كل دفعة عمل مكتملة — أي النمط المتّبع:
  اعمل على الفرع المخصّص → ابنِ وتحقّق → ادفع للفرع → إن طلب المالك "ادمج"
  نفّذ fast-forward إلى `main` وادفع.
- **الخطوة التالية المقترحة لأي جلسة تكمل هذا العمل:**
  1. `git status` للتأكد من عدم فقد أي تعديل غير محفوظ.
  2. أكمل البنود ٢-٦ أعلاه إن لم تكتمل.
  3. `npm run build` للتأكد.
  4. اختبار تشغيلي حقيقي (Playwright/محاكاة) — خصوصاً DayView بمذكرتين
     بنفس اليوم، وحذف المزاج (تأكد ما تبقى مرجع لـ `mood` بأي ملف:
     `grep -rn "mood\|Mood\|MOOD" src/`).
  5. Commit بأسلوب المستودع (رسائل عربية موجزة تشرح "لماذا").
  6. ادفع للفرع `claude/project-review-bugs-busauw`.
  7. لا تدمج لـ `main` إلا إن طلب المالك ذلك صراحةً (كما جرت العادة في
     هذه المحادثة — كان يقول "ادمج" بعد كل مراجعة لنتائج العمل).
  8. **احذف هذا الملف (`HANDOFF.md`) من الجذر قبل آخر دفعة/دمج نهائي** —
     هو ملف عمل داخلي وليس جزءاً من المشروع.
