# CLAUDE.md — دليل العمل في هذا المستودع (مدار)

## سير العمل — أين تكتب التعديلات (الأهم)

**اكتب مباشرةً على `main` وادفع إليه.** هذا مشروع شخصي بمستخدم واحد، والمالك
يريد أن تصل التعديلات مباشرةً للنشر. GitHub Pages ينشر **من `main` فقط**
(`.github/workflows/pages.yml`)، فأي عمل خارج `main` لا يُنشر ولا يظهر.

- **لا تُنشئ فروعاً (branches) ولا Pull Requests** إلا إذا طلب المالك مراجعةً
  صراحةً. دورة الفرع → PR كانت تسبّب تعارضات متكرّرة لأن `main` يتحرّك بين الجلسات.
- **زامِن قبل أن تبدأ وقبل أن تدفع:** `git pull --rebase origin main`. هكذا يبقى
  كل تعديل فوق أحدث `main` ولا تحدث تعارضات أصلاً.
- **ابنِ قبل الدفع:** `npm run build` هو البوابة (لا يوجد CI على المستودع؛
  `next lint` بلا إعدادات ولا يُستخدم). البناء الأخضر = جاهز للدفع.
- بعد الدفع إلى `main`، ينشر workflow الصفحات تلقائياً.
- **إن ألزمتك أدوات الجلسة بفرع عمل رغم ذلك:** أعِد التأسيس على أحدث `main`
  قبل الدفع وأبقِ الفرع قابلاً للدمج فوراً (لا تدع `main` يتقدّم عليه ويتعارض).

## نبذة عن المشروع

- تطبيق PWA شخصي عربي (RTL) اسمه **مدار** لتتبّع: المصاريف، المذكرات، الصلوات،
  القراءة، العادات، والإحصائيات.
- **التقنيات:** Next.js 15 (App Router، `output: "export"` → موقع ثابت)، React 18،
  Zustand (محفوظ في IndexedDB عبر idb-keyval)، Tailwind، Firebase/Firestore
  لمزامنة سحابية اختيارية (نموذج «login-free»: `src/components/sync/`).
- كل الحالة على جهاز المستخدم؛ Firestore للمزامنة فقط.

## الأوامر

- `npm run dev` — خادم تطوير محلي.
- `npm run build` — إخراج ثابت للإنتاج في `out/` (يفحص الأنواع أيضاً).
- النشر: ادفع إلى `main` (تلقائي عبر GitHub Pages).

## أعراف وأفخاخ (حافظ على الاتساق)

- **التواريخ:** دائماً مفاتيح محلية `YYYY-MM-DD` عبر
  `toDateStr`/`parseDate`/`today` في `src/lib/utils.ts`. لا تستخدم
  `toISOString()` لمفاتيح التاريخ (UTC يزيح اليوم في الخليج).
- **الأرقام:** نسّق المبالغ/الأعداد بـ`formatAmount` (أرقام لاتينية عبر
  `ar-SA-u-nu-latn`). لا تستخدم `toLocaleString("ar-SA")` المجرّدة (هندية).
- **المال:** قرِّب عند الحدود بـ`round2`؛ الميزانية اليومية تراكمية
  (`computeDailyBudgetStatus`، مع `carryAdjust` بعد ترحيل الفائض/الراتب).
- **RTL:** الشريط الجانبي الثابت يستخدم `right-0`/`border-l`/`mr-56` الفيزيائية
  عمداً؛ فيما عدا ذلك فضّل الأصناف المنطقية (`ms-`/`me-`/`start`/`end`).
- **الوضع الليلي:** يُقاد بصنف `.dark`؛ خلفيات `bg-*-50/100` الباهتة مُعاد
  تعيينها في `globals.css` — عند إضافة سطح ملوّن جديد أضِف تعيينه الليلي هناك.
- **التنقّل:** المصدر الوحيد هو `src/lib/nav.ts` (يشترك فيه `Sidebar` و`MobileNav`)
  — عدّل المسمّيات هناك.
- **تعديلات المتجر** تختم `lastUpdated` تلقائياً (غلاف `set` في
  `src/lib/store.ts`)؛ استخدم `rawSet` فقط للكتابة المحلية بالجهاز/الترطيب.
